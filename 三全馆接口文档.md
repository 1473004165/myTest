>**安全验证模块**

>登录

~~~properties

URL		 http://xxx.xxx.xx.x:8080/sanquan/
序号	参数名		参数说明	数据类型	可选/必选	备注
1	 studentId    学号	   String	   必选	   如果确定是教务系统登录，表示学号(用户登录请求时须做表单校验)
2	 password	  密码	   String  	   必选	   用户密码（用户登录请求时须做表单校验）
3	 method		登录方式	 Boolean	 必选	     true：用户名密码登录;   false：游客登录
~~~

~~~json
接口名称: 用户登录
url: http://xxx.xxx.xx.x:8080/sanquan/login
请求格式: {"method":true,studentId:"123456789",password:"xxx"}		   #登陆成功后会把用户信息存入session域
请求方式: POST请求
返回值格式1: {
    			loginCode:"0",
    			msg: "用户名或密码错误"
		    }            					    #备注：登陆失败
返回值格式2: {
    			loginCode:"1",
    			name:"熊三",
    			image:"http://xxx.xxx.xx.x:8080/xiongsan.jpg"
		    }								   #备注：登陆成功

返回值说明：

    参数名	 		参数类型		参数说明	必有/未必有	备注
    loginCode	   String		 登录结果状态	    必有	   1:登录成功； 0:代表失败  2:游客登录成功
    msg	   		   String		 请求结果消息	   未必有	  登陆失败时返回的错误信息
    image		   String		 用户的头像		必有	   初始值和游客头像一样，用户修改后使用自己的头像




接口名称: 游客登录
url: http://xxx.xxx.xx.x:8080/sanquan/login
请求格式: {"method":false}		   
请求方式: GET请求
返回值格式: {
    			loginCode:"2",
    			name:"游客",
    			image:"http://xxx.xxx.xx.x:8080/visitor.jpg"          #游客头像资源固定
		    }

返回值说明：

    参数名	 		参数类型		参数说明	必有/未必有	备注
    loginCode	   String		 登录结果状态	    必有	   1:登录成功； 0:代表失败  2:游客登录成功
    image		   String		 游客的头像		必有	   游客登录时头像指定唯一资源


~~~



>**微信预定模块**



~~~properties
URL		 http://xxx.xxx.xx.x:8080/sanquan/book
序号	参数名			参数说明			数据类型	可选/必选	备注
1	 startTime		开始时间	 		  date		 必选		预定自习室开始时间
2	 endTime		结束时间	 		  date		 必选		预定自习室结束时间
3	 roomId			自习(会议)室id	 	Integer	     必选	    预定的自习室的id
4	 emptyId		自习室空闲座位id	   Integer	    必选 	   发起预定请求选中的座位号id
5	 time	        用户预定的总时长	    Long	   必选	  用户预定的使用总时长，来自服务器计算。生成最终订单使用
6	 flag		    标识				  Integer	   必选	  0：自习室；	1：会议室
7	 sitId		    预约的自习室座位id	   Integer	   必选	  续订单人自习室时生效(与预订成功前选择的emptyId等值)
~~~
